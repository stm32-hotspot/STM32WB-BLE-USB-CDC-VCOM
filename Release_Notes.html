Purpose

STM32WB-BLE-USB-CDC-VCOM example demonstrates how to use the BLE component alongside the USB peripheral for point-to-point communications. The application runs the BLE component as a GAP Peripheral and GATT Server, and the USB as a Device Communication Class (CDC), this is a typical application on how to use the STM32WBxx USB Device peripheral where the STM32WB MCU behaves as a USB-to-RS232 bridge following the Virtual COM Port (VCP) implementation.

STM32WB-BLE-USB-CDC-VCOM example is being delivered as-is, not necessarily supported by ST.

This example includes the HAL and LL drivers provided in the STM32CubeWB_v1.13.3 package. For further information about these components, see official package.

Main Changes

Initial release of STM32WB-BLE-USB-CDC-VCOM Example

In the STM32WB-BLE-USB-CDC-VCOM example, the project is generated with the STM32CubeMX tool to initialize the system, peripherals and middleware stacks. User can open the provided ioc file in STM32CubeMX to modify the settings and add additional peripherals, to customize his final application, or generate project for different compiler/toolchain; i.e. MDK-ARM
